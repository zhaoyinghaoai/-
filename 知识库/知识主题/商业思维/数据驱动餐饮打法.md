# 数据驱动餐饮打法

> 不卷产品，卷系统。一台收银机+一个小程序=最低成本抓人、最低门槛留人、最高效率赚钱。早餐市场不缺好吃的包子，缺的是方便快捷到了就能拿走。

## 六步框架

| 步骤 | 核心动作 | 数据/系统支撑 |
|------|---------|-------------|
| 1. 门头诱饵 | 大字「1块钱吃早餐」→扫码注册 | 门头不是菜单，是「你能得到什么」 |
| 2. 月卡锁客 | 9.9买30张3元券，每天限1张 | 来4次回本→锁定至少12次消费 |
| 3. 自动发券 | 7天连续消费送蛋、超7天没来送5元券 | 个性化推送=系统替代人工运营 |
| 4. 30秒取餐 | 9成厨房不设座位，扫码即走 | 别人100单你300-500单 |
| 5. 数据选品 | 后台看销量砍滞销品，按日调整备货量 | 算得越准浪费越少 |
| 6. 简单裂变 | 拉新客双方送免费早餐 | 小程序自动跑，100老客→整片写字楼 |

## 关键洞察

### 门头法则
门头不是写菜单的地方——是写「你能得到什么」的地方。大字说价值（1块吃早餐），小字写条件（原价8块）。顾客不是因为「这家有肉包豆浆」进来的，是因为「1块钱划算」进来的。

### 月卡心理机制
9.9元30张券的本质不是打折——是制造「不来就亏了」的心理压力。来4次回本，剩下26天是白嫖机会，顾客会因为「不用就浪费了」而持续消费。这和健身房的年卡逻辑一模一样。

### 效率即护城河
「好吃」是主观的，永远有比你更好吃的。「快」是客观的，30秒取餐是硬指标。在早餐场景中，方便快捷>味道——顾客早上最缺的不是味觉体验，是时间。
